# Conceptual Model

### Initial Model

The intital model that was designed only compensated for an owner and cars. Then having cars have a maintenance history.

Owner uses a unique identifier of id that has a relation with cars as a one to many relation. Then each car has a one to many relation with the history logs.

Owner can add a car.

Owner can add history to a log.

### Conceptual Model

Final inital model has additional featrues of trusted gas station to gas station that has a zero to one relation keeping track of various states of said gas station.

Owner can add a trusted gas station but does not have to.

Owner can add multiple cars or have none also.

Owner can add/remove history from a maintenance log.
